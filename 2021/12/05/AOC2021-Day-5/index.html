After playing games with our giant squid friend today we are going to do some mapping. We have some hydrothermal vents. We need to find where the vents overlap.

The list contains two endpoints of a vent.

Implementation

First we can create a struct for a point and We can use the point struct to create a line (vent) struct

As our today’s puzzle is pretty straightforward we can implement a few methods easily. First we can think about parsing the input. The input consist of lines like this 0,9 -> 5,9 the starting point is 0,9 and ending point is 5,9. Lets write the functions.

As for the first part we need to get all the points inside a straight line
we can write a function to check if a line is straight or not

If the line straight we need to get the points covered by the line.

Now we have all the functions necessary to solve part 1. For the grid I am creating a HashMap of points and boolean. For creating a HashMap of point we need to derive PartialEq,Eq and Hash traits for our struct point we can do that just by typing #[derive(traits)] before the struct like this.

We can just derive like this because the elements of the struct (i32) have the trait already implemented.

We traverse through the list to create lines. Then we iterate through the lines (redundant I know) and if the line is straight we fetch all the points covered by the line. We put all the points in the hashmap as false. When entering the points in the hashmap if we find that the point exists in the hashmap as false that means it is overlapping we can just change the value to true and if we find it the next time we will ignore because we just have to consider at least 2 lines overlapping if there is more we do not care.
