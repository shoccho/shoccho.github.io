Advent of Code 2021 Day 3

Shoccho
2021-12-03
AOC, Rust

Continuing to day 3 of our journey through AOC while learning rust. Today was in honest words “frustrating” . With a little spike in difficulty and handling bits not being my strong suit I had to spend some extra effort to solve today’s puzzles.

Let’s check today’s problem

We have to analyze a diagnostic report that is generated by our submarine.
The problem is the report is in binary.

Sample report:

Our first task is to find out the power consumption which can be calculated by finding out gamma rate and epsilon rate and multiplying them together.
To find out the gamma or the epsilon rate we need to find the most common bit for gamma and least common bit for epsilon from the report for each index. for example for the dummy input in the first index the most common bit is ‘1’ and least common is ‘0’ and for the second index the most common bit is ‘0’ and least common is ‘1’ subsequently the most common value of the third, fourth, and fifth bits are 1, 1, and 0, respectively, and so the final three bits of the gamma rate are 110.
So, the gamma rate is the binary number 10110, or 22 in decimal and the epsilon rate is the binary number 01001, or 9 in decimal.

My final approach

Now I have already solved both of the puzzles already so I know what I have to do and improve it.

I wrote a function to get the most common bit and the least common bit at a particular index from some lines of binary string.

I think it can be improved a lot but for now it works fine for me.
Now we can use this function to calculate the gamma and epsilon rates by running a loop and calling this method for each index.

We get a (0,1) or a (1,0) tuple from the process_bits function; the first one is most common so it goes to gamma and the second one is the least common bit so we add it to epsilon. And I am just shifting the bits by 1 position and adding 1 bit to the rates so we can get the decimal value directly for the gamma rate and the epsilon rate.

This solves part 1 of day 3.

Now let’s tackle part 2

We now have to find the most common bit and least common bit as before but now we have to filter the binary string list. This is a little tricky as the question already says. If we read this section from the question

We understand that we need 2 copies of the input and filter out each of them separately for Oxygen and CO2 rating.
The thing I missed when I first implemented it was for the 2nd index and later on we need to consider the most common or least common bit from the new filtered input not the whole input.

So first I copied the input bits into o2 and co2 for each of the rating we need to calculate and just like before we pass the input into process_bits function at first and get most & least common bit and for o2 we use the most common bit to filter out the input and save the filtered input to the variable o2 so that we can filter the remaining binary strings.

For co2 we do the same thing but with the least common bit.

If o2 or co2 input has only one binary string remaining we don’t need to filter so we need that if condition.

The i32::from_str_radix(str,b) functions takes a string ‘str’ and tries to convert str into i32 assuming it is in base ‘b’ here we give the filtered out binary string for co2 and o2 and give the base as 2 to convert binary string as i32.
