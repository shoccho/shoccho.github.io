    AOC, Rust

    Now doing day 6 of Advent of code we have to count the number of Lantern fishes.
    Our problem is very simple today. Each Lantern fish creates a new replica of it each 7 days.
    However this doesn’t necessarily mean that all of the lantern fishes makes the replica on the same day. On the starting day we have a list of how many days are left from a lantern fishes 7-day cycle (this is our input)

    This list means that the first fish has an internal timer of 3, the second fish has an internal timer of 4, and so on until the fifth fish, which has an internal timer of 2.

    One other thing is that for the new fish the internal timer is set to 8 instead of because to replicate for the first time it needs two extra days. Think of it as maturing. After maturing it can replicate after 6 days as usual.

    Solving by simulating

    The puzzle is pretty simple you can outright do some clever math stuff and get the correct result or you can try to simulate it. As for part 1 we just have to find out how many Lantern fishes there are after 80 days lets try to simulate the whole process.

    To simulate the situation First we need to create a struct for lantern fish

    Now we have our fish struct we have a days field which contains how many days are left till next replication. I had also added a boolean field called mature but didn’t actually use it later.
    Now as each day passes each lantern fishes days field will decrease by one and when it reaches 0 it will reset and back to 6 create a new lantern fish.
    lets implement the methods for simulating this.

    Now we can parse our input and simulate for 80 days. My idea was to call age on each fish and if it creates a new fish add this to a new child fishes vector after each day add the child fishes to the vector of all fishes.

    This part is pretty easy. But we can’t do this for part 2 because in part 2 we have to do the same thing but for 256 days. And simulating this for 256 isn’t possible by this approach.

    Cheating

    I wouldn’t call this approach cheating but after simulating the first part it feels like it.

    Now we don’t care about the fishes struct at all. We can create an array of days. on each day a when a new fish will be created we can increase the value in the array as at index day.
    For example we have 2 fishes with starting day 2 and 1. so at start we have 2 fishes and on day 1( counting from 0) we will get another fish. For the new fish we will get another fish in day 10 (spawned on day 1 plus 9 days to create new fish as this is the first time) and for the old fish we will get another fish in day 8 (counter reset on day 1 and 7 days for next replication). By this approach we can run it for day 256 and check how many fishes are spawned on day 256 and add the number of initial fishes. we will get the answer.

    I used u128 because for me the answer was very large 1728611055389 so I thought hey lets use as much memory as I can. It is running on my machine after all.

    This approach is also not completely optimized. This is what I could understand and implement there are better ways to do this. It might even be possible to do this in O(1) time, although I am not sure.
